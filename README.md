# Stellar Life

宇宙へ行く人生すごろくのパーティゲーム。地球で普通の人生を始め、学校・仕事・研究・起業・投資を経て、宇宙ステーション、月、火星、そして深宇宙のフロンティアを目指す。

勝利条件は単純なゴール順ではなく **最終スコア**。資金だけでは勝てず、夢・技術・名声・仲間・宇宙貢献度・達成した人生目標がすべて勝敗に効く。**最も伝説的な人生を歩んだプレイヤー**が勝者。

## 特徴

- 2〜8人対応のターン制すごろく
- スマホ最適化 / 1試合およそ15〜30分
- ドット調・ネオン・レトロSFのUI（黒背景、絵文字なし）
- 直接攻撃ではなく、人生イベントと宇宙開発トラブルで盛り上がる
- 全体イベント / ランダム事故 / 燃料ゲートによるルート進行 / 交渉系の選択イベント
- Next.js (App Router) + Supabase Realtime

## ステータス

`資金 / 技術 / 健康 / 名声 / 燃料 / 仲間 / 夢 / リスク` の8種。燃料は宇宙への各ゲート通過に必要、リスクは事故率と最終減点に影響する。

## ルート

`地球の人生 → 低軌道 → 月ルート → 火星ルート → 深宇宙` の順に盤面が宇宙へ広がる。各セグメントの入口には燃料ゲートがあり、燃料が足りないと手前で補給して待機する（恒久的な脱落はなし＝パーティ向け）。

## セットアップ

```bash
npm install
cp .env.example .env.local   # Supabase の URL と publishable key を記入
npm run dev
```

Supabase 側では `supabase/schema.sql` を SQL エディタで実行する。`rooms` テーブルの
`state` jsonb にルーム全体（ロビーの座席 + 進行中のゲーム）が入り、`version` で
楽観的同時実行制御を行い、Realtime の UPDATE 購読で全クライアントへ同期する。
これは StellarBurst のルームモデルと同じ構造なので、同一プロジェクトで両方を運用できる。

`.env.local` が未設定の場合は **ローカルモード** で動作し、1台の端末で全員ぶん +
CPU を操作できる（バックエンド不要で開発・デモ可能）。

## アーキテクチャ

- `lib/game/` — ネットワーク非依存の純粋なゲームエンジン。`engine.ts` の
  `applyAction(state, action)` がすべての遷移を担い、乱数は `GameState.rng`
  （mulberry32 のシード値）を通すので完全に決定的。盤面 `board.ts`、イベント
  `events.ts`、スコア `score.ts`、CPU `cpu.ts`。
- `lib/room.ts` — ロビー + ゲームを包む純粋なルームリデューサ。
- `store/useRoom.ts` — Supabase Realtime 購読と楽観的更新、CPU 進行（ホスト主導）。
  Supabase 未設定時はローカル状態で同じリデューサを回す。
- `app/`, `components/` — UI。

エンジンは純粋なので `lib/game/engine.test.ts` で盤面・進行・スコア・フルゲーム
完走までを単体テストしている。
